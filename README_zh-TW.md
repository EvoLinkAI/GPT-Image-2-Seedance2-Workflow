## 🎥 分鏡工作流程

<a id="case-1"></a>

### 案例 1：[標準分鏡 → 影片](https://x.com/kiyoshi_shin/status/2047133524403400847)（by [@kiyoshi_shin](https://x.com/kiyoshi_shin)）

這是最常見的工作流程。先用 GPT Image 2 生成分鏡畫面，再交給 Seedance 2.0 製作動畫。非常適合宣傳影片、短篇劇情與動畫 OP。

<table><tr>
<td align="center"><a href="https://evolink.ai/gpt-image-2-prompts?utm_source=github&utm_medium=picture&utm_campaign=gptimage2-x-seedance2"><img src="images/storyboard_case1/output.jpg" width="400" alt="分鏡輸出畫面"></a></td>
<td align="center"><video src="https://github.com/user-attachments/assets/ac25fc3d-b6cb-4149-a8ba-e7e10c5b1faa" width="400" controls></video></td>
</tr></table>

**步驟：**

1. 向 GPT Image 2 描述你的場景並生成分鏡圖
2. 在 Seedance 2.0 的 Image-to-Video 模式中匯入分鏡圖
3. 匯出各段片段後，在剪輯軟體中組裝

**GPT Image 2 提示詞：**

```text
Create a 6-panel storyboard for a 15-second brand promotional video. Label each panel with a shot description.
Style: cinematic, cool color tone, widescreen 16:9.
Content: the journey of a product from factory to the customer's hands.
```

**Seedance 2.0 提示詞：**

```text
Cinematic brand advertisement, slow camera push-in, product centered in frame, warm side lighting, soft background blur, no people, 3 seconds.
```

> [!NOTE]
> 建議將分鏡圖輸出為 16:9，避免 Seedance 自動裁切。幀率設定為 24fps 以符合電影常用標準。每格分鏡請盡量簡潔，內容越單純，動作輸出通常越準確。

<a id="case-2"></a>

### 案例 2：[3×3 宮格分鏡法](https://x.com/servasyy_ai/status/2047198012750143999)（by [@servasyy_ai](https://x.com/servasyy_ai)）

這是社群發現的一個關鍵技巧：先把所有分鏡格子組成單張 3×3 宮格圖片，再匯入 Seedance，失敗率會明顯低於逐幀分別匯入。

<table><tr>
<td align="center"><a href="https://evolink.ai/gpt-image-2-prompts?utm_source=github&utm_medium=picture&utm_campaign=gptimage2-x-seedance2"><img src="images/storyboard_case2/output.jpg" width="400" alt="3×3 宮格分鏡輸出"></a></td>
<td align="center"><video src="https://github.com/user-attachments/assets/00f32388-a17b-4b9c-8da3-1956436ce91b" width="400" controls></video></td>
</tr></table>

**步驟：**

1. 要求 GPT Image 2 把所有分鏡格輸出為單張 3×3 宮格圖
2. 將整張宮格圖作為單一圖片匯入 Seedance 2.0
3. Seedance 會把這個宮格視為動作序列，生成連續影片

**GPT Image 2 提示詞：**

```text
Generate a single 3×3 storyboard grid image (9 panels total) showing the following continuous action:
[describe your scene here]
Requirements: each panel is clean and self-contained, character positions are consistent across panels, background is consistent, no text labels or annotations.
Output as a single image with all 9 panels arranged in a grid.
```

**Seedance 2.0 提示詞：**

```text
[Describe the motion and style. Example: Japanese full-color animation, fast cuts, high frame count, 24fps, dark fantasy anime OP style, intense battle scenes.]
```

> [!NOTE]
> **使用前請先替換方括號中的內容。** 這種方法之所以有效，是因為 Seedance 會從單張圖片中分析動作意圖。宮格圖能提供方向性參考，相比多張獨立圖片，更容易產生連貫動作。

<a id="case-10"></a>

### 案例 10：[多幀參考 → 快剪影片](https://x.com/heygentlewhale/status/2047969137969004946)（by [@heygentlewhale](https://x.com/heygentlewhale)）

向 Seedance 2.0 傳入一張包含多個參考幀的分鏡圖，並指示它依序跟隨場景。模型會將幀的位置解讀為場景提示，直接輸出連貫的快剪效果——無需手動拼接。

<table><tr>
<td align="center"><video src="images/storyboard_case10/output.mp4" width="400" controls></video></td>
<td align="center"><video src="images/storyboard_case10/storyboard_ref.mp4" width="400" controls></video></td>
</tr></table>

**步驟：**

1. 在 GPT Image 2 中生成多格分鏡圖（12 幀，3×4 或 4×3 網格）
2. 在 Seedance 2.0 中以該分鏡圖作為參考圖上傳
3. 撰寫包含幀數與剪輯風格的序列提示詞

**GPT Image 2 提示詞：**

```text
Create a 12-panel storyboard grid for a [N]-second [genre] film:
- 4 columns × 3 rows, left-to-right, top-to-bottom reading order
- Each panel: [shot type] + [action description]
- Location: [setting], Time: [day/night], Mood: [atmosphere]
- Consistent character design and scene across all panels
- No text labels, no panel borders
Output as a single image.
```

**Seedance 2.0 提示詞：**

```text
Follow the storyboard sequence of the 12 reference frames in image1, edited as a fast-cut memory montage.
[Describe visual style — example below:]
A nostalgic romance film set in 1990s Singapore, shot on 35mm film in Kodak Portra 800 style.
Soft grain, dreamy blur, warm highlights, and slight color shifts create a vintage cinematic atmosphere.
```

**通用序列提示詞（via [@ai_gezgini](https://x.com/ai_gezgini/status/2047349122315805016)）：**

```text
Use this storyboard to generate a video, follow the scene order, keep transitions smooth,
and preserve cinematic lighting and pacing.
[Add any extra visual details you want.]
```

> [!NOTE]
> 此提示詞適用於多種風格——將風格描述替換為科幻、恐怖、紀錄片或其他任何形式皆可。關鍵句是 `follow the storyboard sequence of the [N] reference frames`——它告訴 Seedance 將幀的位置視為時間軸，而非單張構圖。

**步驟：**

1. 使用 GPT Image 2 生成角色三視圖（正面 / 側面 / 背面）
2. 若有需要，可另外生成裝備圖，再整合成完整裝備版三視圖
3. 以三視圖為參考生成分鏡畫面
4. 把分鏡匯入 Seedance 2.0 製作動畫

**GPT Image 2 提示詞：**

```text
Create a character three-view sheet (front, side, back views) for the following character:
[character name], [hair color], [eye color], [outfit description], [body type].
Style: Japanese anime illustration, clean linework, flat color, white background.
All three views must maintain consistent proportions and design details.
```

**Seedance 2.0 提示詞：待機動作**

```text
Japanese full-color anime style, character in natural idle breathing animation, subtle hair movement, 24fps, seamless loop.
```

**Seedance 2.0 提示詞：戰鬥動作**

```text
Japanese full-color anime style, high-speed cuts, high frame count, 24fps, dark fantasy anime OP style. Protagonist faces a giant creature — a sequence of high-impact combat scenes.
```

### 案例 4：[動畫 OP 風格影片](https://x.com/Toshi_nyaruo_AI/status/2047216971184546231)（by [@Toshi_nyaruo_AI](https://x.com/Toshi_nyaruo_AI)）

用 GPT Image 2 建立場景設定圖，再讓 Seedance 2.0 自由發揮動畫效果。對比「受分鏡約束」與「純提示詞自由生成」兩種輸出，有助於你判斷每個鏡頭該採用哪種方法。

<td align="center"><a href="https://evolink.ai/gpt-image-2-prompts?utm_source=github&utm_medium=picture&utm_campaign=gptimage2-x-seedance2"><img src="images/character_case4/output0.jpg" width="280" alt="動畫 OP 輸出 1"></a></td>

**步驟：**

1. 使用 GPT Image 2 生成一張細節完整的場景設定圖
2. 匯入 Seedance 2.0，搭配簡短的動作提示詞
3. 可額外比較兩種版本：一種嚴格依分鏡控制，另一種讓 Seedance 自由動畫化

**GPT Image 2 提示詞：**

```text
Create a scene setting illustration for a dark fantasy anime:
Location: [describe location], Time: [day/night], Atmosphere: [mood].
Style: Japanese anime production art, high detail, cinematic composition.
Character: [character name and appearance]. Fix this visual design across all panels.
```

**Seedance 2.0 提示詞：**

```text
Japanese full-color anime, fast cuts, high frame count, 24fps. Dark fantasy anime OP style. Epic battle between protagonist and massive supernatural creatures. High-impact sequence of scenes. Only [character name] appears.
```

> [!NOTE]
> 當 Seedance 自由生成動畫時（沒有分鏡參考），畫面通常會更有動態感，但與原始圖片的一致性可能較低。關鍵角色鏡頭建議使用分鏡控制，動作段落則可交給自由動畫處理。

### 案例 12：[Claude Code × 角色設定圖 → 動畫](https://x.com/old_pgmrs_will/status/2045091769180914019)（by [@old_pgmrs_will](https://x.com/old_pgmrs_will)）

先用 Claude Code 撰寫世界觀設定與角色背景，再將結構化描述傳給 GPT Image 2 生成角色關鍵視覺，最後以 Seedance 2.0 製作動畫。對開發者友善的原創 IP 創作工作流程。191 讚 / 7K 觀看。

<table><tr>
<td align="center"><a href="https://evolink.ai/seedance2?utm_source=github&utm_medium=picture&utm_campaign=gptimage2-x-seedance2"><img src="images/character_case12/output.jpg" width="400" alt="Claude Code + GPT Image 2 角色關鍵視覺"></a></td>
</tr></table>

**步驟：**

1. 用 Claude Code 起草世界觀筆記與結構化角色設定（姓名、外貌、個性、背景）
2. 將角色設定直接傳入 GPT Image 2，生成關鍵視覺或角色設定圖
3. 以關鍵視覺作為參考圖匯入 Seedance 2.0，進行動畫化

**Claude Code → GPT Image 2 交接提示詞：**

```text
Based on the following character spec, generate a key visual for [character name]:
[Paste Claude Code output here — name, appearance, outfit, world setting, mood]
Style: [anime / cinematic illustration / game art], [color palette].
Fix this character design — it will be used as a reference across all subsequent images.
```

**Seedance 2.0 提示詞：**

```text
Japanese full-color anime style, character in natural idle breathing animation,
subtle hair and clothing movement, 24fps, seamless loop.
[Or: high-speed cuts, 24fps, dark fantasy anime OP style — protagonist in opening sequence.]
```

> [!NOTE]
> Claude Code 輸出的是結構化文字——角色設定、場景描述、對話大綱——這些作為詳細提示詞傳入 GPT Image 2 效果很好。這條流程線對原創故事 IP 尤其有效：在程式碼中構建世界觀，在 GPT Image 2 中將其視覺化，在 Seedance 中製作動畫。

**步驟：**

1. 向 GPT Image 2 描述你的 App 概念與設計語言
2. 生成 3 到 5 張關鍵 UI 截圖（首頁、功能頁、個人頁等）
3. 依照使用者流程排序後，匯入 Seedance 2.0
4. 匯出展示影片並發布，觀察市場反應

**GPT Image 2 提示詞：**

```text
Design [N] UI screenshots for a "[app concept]" app:
1. [Page 1 name and description]
```

## 💡 技巧與方法

<a id="consistency-guide"></a>

### 一致性指南

在 GPT Image 2 與 Seedance 2.0 的流程中，最常見的挑戰就是維持視覺一致性。下面這些方法分別對應不同層級的問題。

**產品圖一致性**

Seedance 中產品變形的根本原因，在於動態補幀過程會重寫細節，像是 logo、材質與表面紋理都可能被修改。

解法：
- 在 Seedance 提示詞中加入 `keep the product appearance completely unchanged, camera movement only, no rotation`
- 優先選擇鏡頭運動（推近、拉遠）而不是主體運動，讓產品保持靜止，只移動鏡頭
- 將片段控制在 3 秒以內，片段越短，累積變形通常越少

**角色一致性**

- 先生成角色三視圖，作為後續所有分鏡的視覺錨點
- 每一格分鏡提示詞都保留簡短角色描述（髮色、服裝、體型）
- 避免在同一段片段中頻繁切換角色視角

**場景一致性**

當你在 GPT Image 2 中生成多格分鏡時，建議在提示詞最前面固定場景參數：

```text
Scene setting: [location], [time of day], [lighting direction], [fixed background elements].
Maintain this scene setting unchanged across all panels.
```

<a id="troubleshooting"></a>

### 疑難排解

**Seedance 內容審核攔截**

原因：圖片包含被判定為敏感的內容（例如寫實暴力、某些姿勢的人臉）。
解法：改用動畫或插畫風格，或從提示詞中移除人物描述。

**輸出動作混亂**

原因：分鏡圖過於複雜，Seedance 無法判斷主要動作方向。
解法：將分鏡簡化為單一主體與單一清楚動作，減少背景元素。

**產品圖變形**

請參考上方「一致性指南 → 產品圖一致性」段落。

**平台輸入格式需求**

| 平台 | 建議輸入尺寸 | 支援格式 | 檔案大小上限 |
| :---: | :---: | :---: | :---: |
| Hailuo | 1280×720 或 720×1280 | JPG / PNG | 10 MB |
| Higgsfield | 1920×1080 | PNG | 20 MB |
| HitPaw | 任意比例 | JPG / PNG / WEBP | 15 MB |
